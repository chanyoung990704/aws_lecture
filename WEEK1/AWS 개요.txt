클라이언트 - 서버 모델
	클라이언트 : 사람이 컴퓨터 서버에 요청을 보내기 위해 상호 작용하는 도구
	서버 : 클라이언트의 요청을 처리하고 클라이언트에게 응답을 보내주는 것


AWS의 이점
	필요할 때 서버 스케일 조절 가능 ( 사용량에 따라 비용 최적화 )
	선행 비용을 가변 비용으로 대체
	데이터 센터 운영 및 유지 관리 비용 투자 불필요
	용량 추정 불필요
	속도 및 민첩성 향상
	몇 분 만에 전 세계에 배포
	


온디맨드식 제공과 클라우드 배포의 차이점
	온디맨드식 제공 : aws가 사용자에게 필요한 리소스를 필요한 순간에 전달
	클라우드 컴퓨팅 배포 모델
		클라우드 기반 배포
			애플리케이션 모든 부분 클라우드에서 실행
			클라우드에서 애플리케이션 설계 및 빌드, 마이그레이션			
		온프레미스 배포 ( 기존의 방식 )
			가상화 및 리소스 관리 도구 사용하여 리소스 배포
			애플리케이션 관리 및 가상화 기술 사용하여 리소스 활용도 최대화
			
		하이브리드 배포 ( 클라우드와 온 프레미스 둘 다 사용 )
			클라우드 기반 리소스를 온프레미스 인프라에 연결
			클라우드 기반 리소스를 레거시 IT 애플리케이션과 통합



종량 과금제 모델 요약
	종량 과금제 : 사용한 시간 or 사용량 만큼 비용 지불

Amazon Elastic Compute Cloud (Amazon EC2)
	EC2 : 가상화된 서버에 접속
	안전하고 크기 조정이 가능한 컴퓨팅 용량 제공
	인스턴스를 프로비저닝하고 시작 가능
	워크로드 실행 완료했다면 인스턴스 사용 중지 가능
	멀티 테넌시 : 하이퍼바이저가 가상 머신끼리 서로 물리적인 리소스를 공유하도록 책임 그래서 여러 가상머신이 기본적인 하드웨어 공유하는 것
	
EC2 인스턴스 유형
	범용 인스턴스
		컴퓨팅, 메모리, 네트워크 리소스를 균형있게 제공
	컴퓨팅 최적화 인스턴스
		고성능 프로세서 활용 애플리케이션에 적합 ex) 웹 서버, 애플리케이션 서버, 게임 서버
	메모리 최적화 인스턴스
		대규모 데이터 세트를 처리하는 워크로드를 위한 빠른 성능 제공
	엑셀러레이티드 컴퓨팅 인스턴스
		하드웨어 엑셀러레이터 또는 코프로세서를 사용한다. 부동 소수점 수 계산, 그래픽 처리, 데이터 패턴 일치 작업
	스토리지 최적화 인스턴스
		로컬 스토리지의 대규모 데이터 세트에 대한 순차적 읽기 및 쓰기 엑세스가 많이 필요한 워크로드를 위해 설계
EC2 요금
	온디맨드 인스턴스
		중단할 수 없는 불규칙한 단기 워크로드가 있는 애플리케이션에 매우 적합
		사용한 컴퓨팅 시간에 대해서만 비용 지불
	Amazon EC2 Savings Plans
		사용량 약정을 통해 비용 절감
	예약 인스턴스
		계정에서 온디맨스 인스턴스 사용할 때 적용되는 결제 할인 옵션
	스팟 인스턴스
		시작 및 종료 시간이 자유롭거나 중단을 견딜 수 있는 워크로드에 적합
EC2 확장
	확장성을 위해서는 필요한 리소스만으로 시작하고 수요에 따라 자동으로 대응해야 함
	Amazon EC2 Auto Scaling
		변화하는 애플리케이션 수요에 따라 인스턴스를 자동으로 추가 및 제거
		동적 조정 / 예측 조정을 통해 빠르게 조정 가능
		규모 조정
			최소 용량 : Auto Scaling 그룹을 생성한 직후 시작되는 Amazon EC2 인스턴스 수
			희망 용량 : 원하는 인스턴스 수 만큼 확장
			최대 용량 : 수요 증가에 대응하여 확장하도록 인스턴스 수 증가
Elastic Load Balancing
	애플리케이션 트래픽을 여러 리소스에 자동으로 분산하는 aws 서비스 ( 대기가 적은 인스턴스로 보냄 )
	Auto Scaling 그룹으로 들어오는 모든 웹 트래픽의 단일 접점 역할
		=> 가장 먼저 라우팅

메시징 및 대기열
	모놀리식 애플리케이션 및 마이크로 서비스
		모놀리식 애플리케이션 : DB, 서버, 사용자 인터페이스 등 구성 요소들이 밀결합된 애플리케이션
			=> 한 구성 요소에서 장애가 발생하면 다른 곳까지 영향 끼침
		마이크로 서비스 : 애플리케이션 구성 요소가 소결합 ( 메시징 및 대기열을 활용해 구현 )
			=> 한 구성 요소에서 장애가 발생하더라도 다른 구성 요소들은 서로 통신
		
추가 컴퓨팅 서비스
	EC2 실행 과정
		1. 인스턴스(가상서버)를 프로비저닝
		2. 사용자 코드 업로드
		3. 애플리케이션이 실행되는 동안 인스턴스 관리
	서버리스 컴퓨팅
		코드가 서버에서 실행되지만 이 서버를 프로비저닝 하거나 관리할 필요가 없음
	AWS Lambda
		1. 코드를 Lambda에 업로드
		2. AWS 서비스, 모바일 애플리케이션 또는 HTTP 엔드포인트 같은 이벤트 소스에서 트리거되도록 코드 설정
		3. Lambda는 트리거된 경우에만 코드 실행
		람다는 사용한 컴퓨팅 시간에 대한 요금만 지불 => 빠른 처리가 가능한 작업의 경우 적절
	AWS는 컨테이너식 애플리케이션을 빌드하고 실행할 수 있음
		컨테이너 : 애플리케이션의 코드와 종속성을 하나의 객체로 패키징하는 표준 방식 제공	
			=> 환경이 변하더라도 일관성 유지 가능, 규모가 큰 작업에서의 관리 유용
	AWS Fargate
		컨테이너용 서버리스 컴퓨팅 엔진 Amazon ECS(신축하게 구축, 테스트 전용)와 Amazon EKS(대규모 배포)에서 작동
		OS에 대한 완전한 엑세스 권한 사용 가능
AWS 글로벌 인프라
	리전
		데이터를 저장할 수 있는 장소
		다양한 지역의 데이터 센터 존재
		고속 광섬유 네트워크 통해 다른 리전에 연결
		적절한 인증 절차를 거치지 않으면 다른 지역의 리전으로 데이터를 전송 불가
	
		- 데이터 거버넌스 및 법적 요구 사항 준수
		- 근접성 ( 지연시간을 고려해야 함 / 가까운 위치의 리전 선택해야 함 )
		- 기능 가용성 
		- 요금 ( 지역에 따라 요금이 다를 수 있다 )
	AWS 데이터 센터
		전 세계 여러 곳에 위치
		가용 영역
			여러 곳에 위치한 리전에서 인스턴스 실행하게 함
			대규모 재난 대비 가능
			
	 
		
엣지 로케이션
	Amazon CloudFront가 더 빠른 콘텐츠 전송을 위해 고객과 가까운 위치에 콘텐츠 사본을 캐시하는 데 사용하는 사이트
	
AWS 서비스와 상호 작용하는 방법
	AWS Management Console
		브라우저 기반
		서비스 처음 시작해 관련 지식을 쌓을 때 유용 ex) 테스트 환경, 모니터링, 기술과 무관한 리소스를 사용하는 작업		
	AWS 명령줄 인터페이스(CLI)
		머신의 터미널을 사용하여 API 호출
		작업을 스크립트로 만들고 반복 가능
		자동화 시스템 구축 가능
	소프트웨어 개발 키트(SDK)
		다양한 프로그래밍 언어를 통해 AWS 리소스와 상호 작용 가능
	AWS Elastic Beanstalk
		사용자가 코드 및 구성 설정을 제공하면 알아서 환경 구성
		인프라가 아닌 비즈니스 애플리케이션에 집중하게 도와줌
		용량 조정 / 로드 밸런싱 / 자동 조정
	AWS CloudFormation
		인프라를 코드로 취급
		개별적으로 리소스를 프로비저닝하는 대신 코드 줄을 작성하여 환경 구축 가능
		스토리지 / 데이터베이스 / 분석 기계학습
		자동화된 프로세스

		
네트워킹
	Amazon Virtual Private Cloud(VPC)
		AWS 클라우드의 격리된 세션을 프로비저닝할 수 있다.
		격리된 섹션에서는 사용자가 정의한 가상 네트워크에서 리소스를 시작할 수 있다.
		여러 서브넷으로 리소스를 구성 가능
	가상 프라이빗 게이트웨이
		비공개 출입구
		VPN 연결 사용
		보호된 인터넷 트래픽이 VPC로 들어오도록 허용

	AWS Direct Connect
		데이터 센터와 VPC 간에 비공개 전용 연결을 설정하는 서비스
		전용 통신 회선을 사용해 지연 방지
	VPC
		서브넷 : 리소스를 그룹화하는 데 사용되는 별개 영역
			퍼블릭 서브넷 : 누구나 엑세스할 수 있어야 하는 리소스 포함
			프라이빗 서브넷 : 프라이빗 네트워크를 통해서만 엑세스할 수 있는 리소스 포함
		VPC 내에서 서브넷은 서로 통신할 수 있다.	
		
		데이터를 요청하면 요청은 패킷으로 전송
		패킷 : 네트워크를 통해 전송되는 데이터의 단위
		패킷은 인터넷 게이트웨이를 통해 VPC로 들어감
		패킷이 서브넷으로 들어가거나 나오려면 먼저 권한 확인이 필요.
		네트워크 ACL
			서브넷 수준에서 인바운드 및 아웃바운드 트래픽 제어하는 가상 방화벽 ( 패킷 권한 확인 )
			사용자 자체 규칙 추가 가능
		상태 비저장 패킷 필터링
			아무것도 기억하지 않고 각 방향으로 서브넷 경계를 통과하는 패킷만 확인
			해당 요청에 대한 패킷 응답이 서브넷으로 반환될 때 네트워크 ACL은 이전 요청을 기억하지 못 함
		보안 그룹
			EC2 인스턴스에 대한 인바운드 및 아웃바운드 트래픽 제어하는 가상 방화벽
			기본적으로 모든 인바운드 트래픽 거부하고 모든 아웃바운드 트래픽 허용
			상태 저장 패킷 필터링
				해당 요청에 대한 패킷 응답이 인스턴스로 반환될 때 보안 그룹이 이전 요청을 기억

글로벌 네트워킹
	Amazon Route 53
		DNS 웹 서비스
		개발자와 비즈니스가 최종 사용자를 AWS에서 호스팅되는 인터넷 애플리케이션으로 라우팅할 수 있는 안정적인 방법 제공
		사용자 요청을 AWS에서 실행되는 인프라에 연결 또한 사용자를 AWS 외부의 인프라로 라우팅할 수 있다.
		단일 위치에서 모든 도메인 이름 관리할 수 있다.
	
스토리지 및 데이터 베이스
-인스턴스 스토어 및 Amazon Elastic Block Store (EBS)
	인스턴스 스토어
		블록 수준 스토리지 볼륨은 물리적 하드 드라이브처럼 동작
		EC2 인스턴스에 임시 블록 수준 스토리지 제공
		물리적으로 EC2 호스트 컴퓨터에 연결되어 인스턴스와 수명이 동일
		인스턴스가 종료되면 스토어 데이터 손실
	EBS
		EC2 인스턴스에서 사용할 수 있는 블록 수준 스토리지 볼륨
		인스턴스가 중지 또는 종료되더라도 연결된 EBS 볼륨의 모든 데이터 사용 가능
	EBS 스냅샷
		증분 백업
		처음 볼륨 백업하면 모든 데이터가 복사 이후 백업에서는 가장 최근 스냅샷 이후 변경된 데이터 블록만 저장 ( 전체 백업과는 다른 개념 )
-Amazon Simple Storage Service (Amazon S3)
	객체 스토리지
		객체는 데이터, 메타데이터, 키로 구성
	S3
		객체 수준 스토리지 제공하는 서비스
		데이터를 버킷에 객체로 저장
		모든 유형의 파일 업로드 가능 / 권한 설정하여 엑세스 제어 가능
	S3 스토리지 클래스
		사용한 만큼 비용 지불
		비즈니스 및 비용 요구 사항에 맞춰 다양한 스토리지 클래스 중 선택 가능
			고려 사항 : 데이터 검색할 빈도 / 필요한 데이터 가용성
		S3 Standard
			자주 엑세스하는 데이터용으로 설계
			최소 3개의 가용 영역에 데이터를 저장
			객체에 대한 고가용성 제공 -> 광버위한 사용 사례에 적합
		S3 Standard Infrequent Access (IA)
			자주 엑세스하지 않는 데이터에 이상적
			스토리지 가격이 보다 저렴하고 검색 가격은 더 높음
		S3 One Zone-Infrequent Access( One Zone-IA )
			단일 가용 영역에 데이터를 저장
			IA보다 낮은 스토리지 가격
		S3 Intelligent-Tiering
			엑세스 패턴을 알 수 없거나 자주 변화하는 데이터에 이상적
			객체당 소량의 월별 모니터링 및 자동화 요금 부과
		S3 Glacier
			데이터 보관용 저비용 스토리지
			객체를 몇 분에서 몇 시간 이내에 검색
		S3 Glacier Deep Archive
			보관에 이상적인 가장 저렴한 객체 스토리지 서비스
			객체를 12시간 이내에 검색
-Amazon Elastic File System(EFS)
	파일 스토리지
		여러 클라이언트가 공유 파일 폴더에 저장된 데이터에 엑세스 가능
		서버가 블록 스토리지를 로컬 파일 시스템과 함께 사용하여 파일 구성
		클라이언트는 파일 경로를 통해 데이터에 엑세스
	 	많은 수의 서비스 및 리소스가 동시에 동일한 데이터에 엑세스해야 하는 사용 사례에 이상적
	EFS
		AWS 클라우드 서비스 및 온 프레미스 리소스와 함께 사용되는 확장 가능한 파일 시스템
		파일을 추가 / 제거하면 EFS가 자동적으로 확장 / 축소
		애플리케이션을 중단하지 않고 온 디맨드로 패타바이트 규모로 확장 가능

EBS 볼륨은 단일 가용 영역 !! EFS는 리전별 서비스 여러 가용 영역에 걸쳐 데이터 저장 !!!

-Amazon Relational Database Service(RDS)
	관계형 데이터 베이스
		데이터가 다른 데이터 부분과 관련된 방식으로 저장
		SQL 사용하여 데이터를 저장하고 쿼리
	RDS
		하드웨어 프로비저닝, 데이터 베이스 설정, 패치 적용 백업과 같은 작업을 자동화하는 관리형 서비스
		다른 서비스와 통합하면 AWS lambda를 사용하여 서버리스 애플리케이션에서 데이터베이스를 쿼리하는 등 비즈니스 및 운영 요구 사항을 총족할 수 있다.
		다양한 보안 옵션 제공
	RDS 데이터 베이스 엔진
		Amazon Aurora / PostgreSQL / MySQL / MariaDB / Oracle Database
	Amazon Aurora
		엔터프라이즈급 관계형 데이터베이스 => 속도 빠름
		데이터베이스 리소스의 안정성 및 가용성을 유지하면서 불필요한 입/출력 작업을 줄여 데이터베이스 비용 절감
-Amazon DynamoDB
	비관계형 데이터베이스
		테이블 생성 / 테이블은 데이터를 저장하고 쿼리할 수 있는 장소
		행과 열이 아닌 구조를 사용하여 데이터를 구성 => NoSQL데이터베이스
		구조적 접근 방식 유형은 키 - 값 페어
			=> 데이터가 항목(키)로 구성되고 항목은 속성(값)을 갖는다.
	서버리스
	자동 조정
-Amazon Redshift
	빅 데이터 분석에 사용할 수 있는 데이터 웨어하우징 서비스
	여러 원본에서 데이터를 수집하여 데이터 간의 관계 및 추세를 파악하는데 도움

-AWS Database Migration Service(DMS)
	데이터 저장소를 마이그레이션 할 수 있는 서비스
	원본 데이터베이스와 대상 데이터베이스 유형이 동일할 필요는 없음
	마이그레이션하는 동안 원본 데이터베이스가 계속 작동하므로 데이터베이스를 사용하는 애플리케이션의 가동 중지 시간을 줄일 수 있다.
	개발 및 테스트 데이터베이스 마이그레이션
	데이터베이스 통합
	연속 복제
-Amazon DocumentDB
	워크로드 지원 문서 데이터베이스 서비스
-Amazon Neptune
	그래프 데이터베이스 서비스
-Amazon Quantum Ledger Database(QLDB)
	원장 데이터베이스 서비스
	모든 변경 사항의 전체 기록을 검토할 수 있다.
-Amazon Managed Blockchain
	블록 체인 네트워크를 생성하고 관리
-Amazon ElastiCache

-Amazon DynamoDB accelerator
	DynamoDB용 인 메모리 캐시 -> 응답시간 마이크로 초까지 향상 가능

보안
	AWS 공동 책임 모델
		AWS 환경을 단일 객체로 취급하지 않음
		AWS는 사용자 환경의 일부분을 책임지고 고객은 다른 부분을 책임짐
		고객 책임 ( 클라우드 내부 보안 )
			고객 데이터 / 플랫폼, 애플리케이션, 자격 증명 및 액세스 관리 / 운영 체제, 네트워크, 방화벽 구성 / 클라이언트 측 데이터 암호화, 서버 측 암호화, 네트워킹 트래픽 보호
		AWS 책임 ( 클라우드 자체 보안 )
			소프트웨어 / 컴퓨팅, 스토리지, 데이터베이스, 네트워킹 / 하드웨어, AWS 글로벌 인프라 / 리전, 가용 영역, 엣지 로케이션
	AWS Identity and Access Management(IAM)
		AWS 서비스와 리소스에 대한 엑세스를 안전하게 관리 
		엑세스 권한 구성 유연성 제공
		AWS 계정 루트 사용자
			모든 AWS 서비스 및 리소스에 대한 전체 엑세스 권한
		IAM 사용자
			AWS 서비스 및 리소스와 상호작용하느 사람 또는 애플리케이션
			이름과 자격 증명으로 구성
			IAM 사용자가 AWS에서 EC2인스턴스 시작, S3 버킷 생성 등 특정 작업을 수행할 수 있도록 허용하려면 필요 권한 부여 필요
			개별 IAM 사용자 생성하는 것 권장
		IAM 정책
			AWS 서비스 및 리소스에 대한 권한을 허용하거나 거부하는 문서
			사용자가 리소스에 엑세스할 수 있는 수준을 사용자 지정 가능
			최소 권한 보안 원칙 준수해야 함
		IAM 그룹
			IAM 사용자 모음 -> 그룹에 정책 할당하면 해당 그룹 모든 사용자에게 권한 부여
		IAM 역할
			임시로 권한에 엑세스하기 위해 수임할 수 있는 자격 증명 ( 일시적 상황에 적합 )
				=> 이전 모든 권한을 포기하고 새로운 권한 획득
		Multi-Factor Authentication
			AWS에 추가 보안 계층 제공

	AWS Organizations
		중앙 위치에서 여러 AWS 계정 통합 및 관리
		조직 생성하면 AWS Organizatios가 조직의 모든 계정에 대한 상위 컨테이너 루트를 자동으로 생성
		서비스 제어 정책(SCP) 사용하여 조직의 계정에 대한 권한을 중앙에서 제어
		조직 단위
			계정을 조직 단위로 그룹화하여 비슷한 비즈니스 또는 보안 요구 사항이 있는 계정을 손쉽게 관리
			OU 정책을 적용하면 OU의 모든 계정이 정책에 지정된 권한 자동 상속

	AWS Artifact
		AWS 보안 및 규정 준수 보고서 및 일부 온라인 계약에 대한 온디맨스 엑세스 제공
		AWS Artifact Agreements
			개별 계정 및 AWS Organizations 내 모든 계정에 대한 계약을 검토, 수락 및 관리 가능
		AWS Artifact Reports
			외부 감사 기관이 작성한 규정 준수 보고서 제공
				=> 표준 및 규정을 준수했음을 검증
	고객 규정 준수 센터
		고객 규정 준수 사례를 읽고 규제 대상 업종의 기업들이 다양한 규정 준수, 거버넌스 및 감사 과제를 어떻게 해결했는지 확인할 수 있다.
		주요 규정 준수 질문에 대한 AWS 답변
		AWS 위험 및 규정 준수 개요
		보안 감사 체크리스트
		감사자 학습 경로 포함

	서비스 거부 공격(Dos)
		사용자들이 웹 사이트 또는 애플리케이션을 이용할 수 없게 만들려는 의도적 시도 ( 단일 공격 )
		ex) 과부하가 걸려 더 이상 응답할 수 없을 때까지 네트워크 트래픽 플러드시킴.
	분산 서비스 거부 공격(DDoS)
		여러 소스를 사용하여 웹 사이트 또는 애플리케이션을 사용할 수 없게 만드는 공격 시작
	AWS Shield
		DDoS 공격으로부터 애플리케이션을 보호하는 서비스
		AWS Shield Standard
			모든 AWS 고객을 자동으로 보호하는 무료 서비스
		AWS Shield Advanced
			상세한 공격 진단 및 정교한 DDoS 공격을 탐지하고 완화할 수 있는 기능 제공하는 유로 서비스
	AWS Key Management Service(KMS)
		저장 상태에서(저장 시 암호화) 그리고 전송되는 동안(전송 중 암호화) 데이터가 안전한지 확인 필요
		암호화 키를 사용하여 암호화 작업 수행
		암호화 키 생성, 관리 및 사용 가능
		키에 필요한 엑세스 제어를 특정 수준으로 선택 가능
			ex) 키를 관리할 수 있는 IAM 사용자 및 역할 지정
	AWS WAF
		웹 애플리케이션으로 들어오는 네트워크 요청을 모니터링할 수 있는 방화벽
		AWS 리소스를 보호하기 위해 웹 ACL(엑세스 제어 목록)을 사용
		요청이 들어오면 웹 ACL에서 구성한 규칙 목록 확인 -> 차단 IP 주소가 아니라면 엑세스 허용
	Amazon Inspector
		자동 보안 평가 수행
		평가를 수행한 후 보안 탐지 결과 목록 제공
	Amazon GuardDuty
		AWS 인프라 및 리소스에 대한 지능형 위협 탐지 기능 제공하는 서비스
		AWS 환경 내의 네트워크 활동 및 계정 동작을 지속적으로 모니터링하여 위협 식별


모니터링 및 분석
	Amazon CloudWatch
		다양한 지표를 모니터링 및 관리하고 해당 지표의 데이터를 기반으로 경보 작업 구성할 수 있는 웹 서비스
		지표를 사용하여 리소스의 데이터 포인트 나타냄
		AWS 서비스가 지표를 전송, CloudWatch가 지표를 바탕으로 그래프 생성
		CloudWatch 경보
			지표의 값이 미리 정의된 임계값을 상회 또는 하회할 경우 자동적으로 작업 수행하는 경보 생성 가능
			ex)CPU 사용률이 특정 임계값 미만일 때
		CloudWatch 대시보드
			단일 위치에서 리소스에 대한 모든 지표에 엑세스할 수 있다.
			EC2 인스턴스의 CPU 사용률, S3 버킷에 대해 실행된 총 요청 수 등 모니터링

	Amazon CloudTrail
		계정에 대한 API 호출 기록
		애플리케이션 및 리소스에 대한 사용자 활동 및 API 호출의 전체 내역 확인 가능
		ex) 이벤트 필터링
		이벤트 레코드는 발생한 작업에 대한 전체 세부 정보를 제공
		CloudTrail Insights
			AWS계정에서 비정상적인 API활동 자동 감지
			ex)평소보다 많은 수 의 EX2 인스턴스
	AWS Trusted Advisor
		AWS환경을 검사하고 AWS 모법 사례에 따라 실시간 권장 사항을 제시
		비용 최적화, 성능, 보안, 내결함성, 서비스 한도 5개 범주에서 결과를 AWS 모범 사례와 비교
			=> 각 범주 검사에 대해 권장 작업 목록 제공
		AWS Trusted Advisor 대시보드
			AWS Management Console에서 Trusted Advisor 대시보드에 엑세스하면 비용 최적화, 성능, 보안, 내결함성, 서비스 한도 범주에서 완료된 검사를 검토

			연략 창구
			모든 AWS 서비스에 대한 전문 지식 제공 -> 솔루션 설계 과정 도움
	AWS Marketplace
		ISV의 소프트웨어 리스팅 수천 개가 포함된 디지털 카탈로그
		AWS에서 실행되는 소프트웨어 검색하고 평가 구매 가능
		각 리스팅에 대해 요금 옵션, 사용 가능한 지원 및 다른 AWS 고객의 리뷰 등 자세한 정보에 엑세스 가능
		솔루션 탐색 사능

마이그레이션 및 혁신
	AWS Cloud Adoption Framework(AWS CAF)
		비즈니스 관점
			IT가 비즈니스 요구 사항을 반영하고 IT 투자가 주요 비즈니스 결과와 연계되도록 보장
			강력한 비즈니스 사례를 설정하고 클라우드 채택 이니셔티브의 우선 순위 지정
		인력 관점
			클라우드 채택을 성공하기 위한 조직 전반의 변화 관리 전략 개발 지원
			조직 구조 및 역할, 새로운 기술 및 프로세스 요구 사항을 평가하고 격차 파악
			교육, 인력 배치 및 조직 변화의 우선 순위 지정
		거버넌스 관점
			IT 전략이 비즈니스 전략에 부합하도록 조정하는 기술 및 프로세스에 중점을 둠
			비즈니스 가치 극대화하고 위험 최소화
			클라우드에서 비즈니스 거버넌스를 보장하는 데 필요한 직원 기술 및 프로세스를 업데이트 하는 방법을 이해
		플랫폼 관점
			클라우드를 기반으로 새로운 솔루션을 구현하고 온프레미스 워크로드를 클라우드로 마이그레이션하기 위한 원칙과 패턴이 포함
			다양한 아키텍처 모델을 사용하여 IT 시스템의 구조와 그 관계를 이해하고 전달
		보안 관점
			조직이 가시성, 감사 가능성, 제어 및 민첩성에 대한 보안 목표를 충족하도록 보장
			AWS CAF 사용하여 조직의 요구 사항에 맞춰 보안 제어의 선택 및 구현 구성
		운영 관점
			비즈니스 이해당사자와 합의된 수준까지 IT 워크로드를 구현, 실행, 사용, 운영 및 복구하는데 도움
			일별, 분기별 및 연간으로 비즈니스를 수행하는 방법 정의
			비즈니스 운영을 반영하고 지원

	마이그레이션 전략
		리호스팅
			애플리케이션을 변경 없이 이전
		리플랫포밍
			리프트 앤 시프트 및 수정
			실질적인 이점을 실현하기 위해 몇 가지 클라우드 최적화 수행
		리팩터링 / 아키텍처 재설계
			클라우드 네이티브 기능을 사용하여 애플리케이션을 설계하고 개발하는 방식을 재구성
		재구매
			기존 라이선스를 Software-as-a-Service 모델로 전환
		유지
			비즈니스에 중요한 애플리케이션을 소스 환경에 유지
		폐기
			더 이상 필요하지 않은 애플리케이션을 제거하는 프로세스

	AWS Snow 패밀리 멤버
		AWs와 고객 간 최대 엑사바이트 규모의 데이터를 물리적으로 이동할 수 있는 물리적 디바이스 모음
		AWS SNOWCONE
			작고 견고하며 안전한 엣지 컴퓨팅 및 데이터 전송 디바이스
		AWS SNOWBALL
			Snowball Edge Storage Optimized
				대규모 데이터 마이그레이션 및 반복 전송 워크플로뿐 아니라 큰 용량이 필요한 로컬 컴퓨팅에 적합
			Snowball Edge Compute Optimized
				기계 학습, 풀 모션 비디오 분석, 분석 및 로컬 컴퓨팅 스택과 같은 사용 사례를 위한 강력한 컴퓨팅 리소스 제공
		AWS SnowMobile
			대용량 데이터를 AWS로 이동하는 데 사용하는 엑사바이트 규모 데이터 전송 서비스

	AWS 서비스를 통한 혁신
		서비스 애플리케이션
			서버리스는 사용자가 서버를 프로비저닝, 유지 관리 또는 관리할 필요가 없는 애플리케이션 의미
			내결함성 또는 가용성을 걱정할 필요 없음
		인공 지능
		기계 학습
			ML모델을 신속하게 빌드, 훈련, 배포하는 데 사용가능한 Amazon SageMaker 제공

클라우드
	AWS Well-Architected 프레임워크
		AWS클라우드에서 안정적이고 안전하며 효율적이고 비용 효율적인 시스템을 설계하고 운영하는 방법 이해하는 데 도움
		운영 우수성
			시스템을 실행 및 모니터링하여 비즈니스 가치를 제공하고 지속적으로 지원 프로세스 및 절차 개선
		보안성
			위험 평가 및 완화 전략을 통해 비즈니스 가치를 제공하는 동시에 정보, 시스템, 자산을 보호하는 능력
		안정성
			인프라 또는 서비스 중단으로부터 복구 / 컴퓨팅 리소스를 동적으로 확보하여 수요 충족 / 잘못된 구성 또는 네트워크 오류 완화
		성능 효율성
			컴퓨팅 리소스를 효율적으로 사용하여 시스템 요구 사항을 충족하고, 수요 변화와 기술 진화에 따라 효율성 유지하는 능력
		비용 최적화
			가장 낮은 가격으로 비즈니스 가치를 제공하도록 시스템을 실행하는 능력
	클라우드 컴퓨팅의 이점
		선행 비용을 가변 비용으로 대체
		거대한 규모의 경제로 얻게 되는 이점
		용량 추정 불필요
		속도 및 민첩성 향상
		데이터 센터 운영 및 유지 관리에 비용 투자 불필요
		몇 분 만에 전 세계에 배포
