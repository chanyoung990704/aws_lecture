클라이언트 - 서버 모델
	클라이언트 : 사람이 컴퓨터 서버에 요청을 보내기 위해 상호 작용하는 도구
	서버 : 클라이언트의 요청을 처리하고 클라이언트에게 응답을 보내주는 것


AWS의 이점
	필요할 때 서버 스케일 조절 가능 ( 사용량에 따라 비용 최적화 )
	선행 비용을 가변 비용으로 대체
	데이터 센터 운영 및 유지 관리 비용 투자 불필요
	용량 추정 불필요
	속도 및 민첩성 향상
	몇 분 만에 전 세계에 배포
	


온디맨드식 제공과 클라우드 배포의 차이점
	온디맨드식 제공 : aws가 사용자에게 필요한 리소스를 필요한 순간에 전달
	클라우드 컴퓨팅 배포 모델
		클라우드 기반 배포
			애플리케이션 모든 부분 클라우드에서 실행
			클라우드에서 애플리케이션 설계 및 빌드, 마이그레이션			
		온프레미스 배포 ( 기존의 방식 )
			가상화 및 리소스 관리 도구 사용하여 리소스 배포
			애플리케이션 관리 및 가상화 기술 사용하여 리소스 활용도 최대화
			
		하이브리드 배포 ( 클라우드와 온 프레미스 둘 다 사용 )
			클라우드 기반 리소스를 온프레미스 인프라에 연결
			클라우드 기반 리소스를 레거시 IT 애플리케이션과 통합



종량 과금제 모델 요약
	종량 과금제 : 사용한 시간 or 사용량 만큼 비용 지불

Amazon Elastic Compute Cloud (Amazon EC2)
	EC2 : 가상화된 서버에 접속
	안전하고 크기 조정이 가능한 컴퓨팅 용량 제공
	인스턴스를 프로비저닝하고 시작 가능
	워크로드 실행 완료했다면 인스턴스 사용 중지 가능
	멀티 테넌시 : 하이퍼바이저가 가상 머신끼리 서로 물리적인 리소스를 공유하도록 책임 그래서 여러 가상머신이 기본적인 하드웨어 공유하는 것
	
EC2 인스턴스 유형
	범용 인스턴스
		컴퓨팅, 메모리, 네트워크 리소스를 균형있게 제공
	컴퓨팅 최적화 인스턴스
		고성능 프로세서 활용 애플리케이션에 적합 ex) 웹 서버, 애플리케이션 서버, 게임 서버
	메모리 최적화 인스턴스
		대규모 데이터 세트를 처리하는 워크로드를 위한 빠른 성능 제공
	엑셀러레이티드 컴퓨팅 인스턴스
		하드웨어 엑셀러레이터 또는 코프로세서를 사용한다. 부동 소수점 수 계산, 그래픽 처리, 데이터 패턴 일치 작업
	스토리지 최적화 인스턴스
		로컬 스토리지의 대규모 데이터 세트에 대한 순차적 읽기 및 쓰기 엑세스가 많이 필요한 워크로드를 위해 설계
EC2 요금
	온디맨드 인스턴스
		중단할 수 없는 불규칙한 단기 워크로드가 있는 애플리케이션에 매우 적합
		사용한 컴퓨팅 시간에 대해서만 비용 지불
	Amazon EC2 Savings Plans
		사용량 약정을 통해 비용 절감
	예약 인스턴스
		계정에서 온디맨스 인스턴스 사용할 때 적용되는 결제 할인 옵션
	스팟 인스턴스
		시작 및 종료 시간이 자유롭거나 중단을 견딜 수 있는 워크로드에 적합
EC2 확장
	확장성을 위해서는 필요한 리소스만으로 시작하고 수요에 따라 자동으로 대응해야 함
	Amazon EC2 Auto Scaling
		변화하는 애플리케이션 수요에 따라 인스턴스를 자동으로 추가 및 제거
		동적 조정 / 예측 조정을 통해 빠르게 조정 가능
		규모 조정
			최소 용량 : Auto Scaling 그룹을 생성한 직후 시작되는 Amazon EC2 인스턴스 수
			희망 용량 : 원하는 인스턴스 수 만큼 확장
			최대 용량 : 수요 증가에 대응하여 확장하도록 인스턴스 수 증가
